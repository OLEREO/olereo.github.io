ZIKU! - 简而全的“软件管理”软件

发表于 2016-06-18   |   分类于 ZIKU!   |     |   阅读次数

虽然写管理软件的人很多，但是貌似写一款“软件管理”的软件还真的是很少，‘这让我产生了编写 ZIKU! 的想法。

ZIKU!的功能不多，基本功能是”软件管理”，额外附带“软件启动器”功能。

和网上各种的软件管家不一样，ZIKU! 并不是一款集”网络软件宝库”，并能推送更新的”贴心管家”。ZIKU! 中的收录的软件信息都是需要我们自己添加的，我们可以添加任何比起软件介绍更加有用的信息。

如果 ZIKU! 只能做到记录一款软件的基本信息的话，那 ZIKU! 就不应该存在，下面列举的 ZIKU! 的其他一些功能。

匹配模式介绍

1.完全匹配 名称中的Firefox
2.匹配名称中文浏览器的 拼音首字母
3.匹配名称中 顺序匹配，但不相连 的文字（示例中，使用 xf 则无法匹配），这个我称呼为“模糊搜索”
4.同时匹配英文和拼音首字母
5.最后一个乍看不管是英文字母还是首字母都没有匹配，但是却在搜索结果中出现，甚至排在了名称以“h”开头的项目前面，位列第一。

其实这是 ZIKU! 为搜索增强添加的一个功能，叫做“搜索缩写”。

在编辑项目的时候，为项目添加一个搜索缩写，能实现 排序靠前 和 搜索别名

例如这里的 “hh” 代表的是 “Firefox” 的中文名 “火狐” 的首字母缩写。

除了可以搜索项目的名称之外，ZIKU! 还支持搜索“项目介绍”。

假设某天我们的健忘症又犯了，但是你却突然想起除了 IE 之外还有一款好用的浏览器，但是项目的名称中也没有写有 浏览器，所以你无法通过上面的方法搜索到该项目。

但是万幸的是，你有在项目的 介绍 中详细的写了关于这款软件的功能，其中就包含了关键字 浏览器 。如果你有选用我们的产品 ZIKU!，就能够解决这一难题。

首先我们在搜索框中输入 j 然后加上一个空格 “ “ ，这表示切换到搜索 “项目介绍” 模式。

父子分类

ZIKU! 的分类只有两个层级，右边是”父分类”，左边”子分类”。

两个层级的分类限制，是”我院(FENLIEZHENG)”经过多日的研讨得出的最佳方案。

因为从狭义来讲软件就两种分类：”系统软件”和”应用软件”。

分类的层级限制不会限制到项目的分类选择，一个项目仍旧可以选择多个不同种类的分类。

目录管理，路径变量

路径变量不是什么新的功能，Windows就自带了一堆环境变量

除了支持Windows的环境变量之外， ZIKU! 还可以自定义路径变量，如果你有用的到的话，这会是一个很有用的功能。而如果你用不到的话，这对于你来讲就是一个摆设。

这是一个很特别的功能，因为这个功能的开始并不是为了软件的管理方面所写的。而是我在开发 ZIKU! 的时候总是为了将一些文件丢到对应的文件夹，以及想着该如何才能快速的将一些从网上下载的东西快速的丢到它本应该呆的目录去。

除此之外和目录管理相关联的一个功能就是路径变量了，如果说目录管理的作用不大的话，那么路径变量则是点睛之笔。

使用路径变量的一个好处就是，当有多个地方使用了同一个主值时，修改该主值之后，无需逐个去修改这些地方。
